Техническое задание: SilenceCut — инструмент автоматического удаления тишины из видео

1. Обзор проекта
SilenceCut — утилита для автоматического удаления тишины из разговорных видео на базе FFmpeg. Поддерживает четыре режима работы: CLI-скрипт, агентный режим, локальный веб-интерфейс и Docker-сервис.

2. Общие требования
2.1 Зависимости

FFmpeg установлен в системе (используется как внешний инструмент через subprocess)
Python 3.10+
Поддержка файлов любого размера (20+ ГБ), без загрузки в память целиком
Входные форматы: MP4, MOV, MKV, AVI, WebM и любые другие, поддерживаемые FFmpeg
Выходной формат: по умолчанию MP4 (H.264 + AAC), настраивается

2.2 Параметры обнаружения тишины (единые для всех режимов)
Все параметры должны иметь значения по умолчанию и быть переопределяемы:
ПараметрОписаниеПо умолчаниюsilence_thresholdУровень тишины в dB (silencedetect noise)-30dBsilence_durationМинимальная длительность тишины для вырезания (сек)0.5padding_startОставить N мс тишины перед речью100mspadding_endОставить N мс тишины после речи150msmin_segment_durationМинимальная длина сегмента речи (сек)0.3speed_up_silenceУскорить тишину вместо вырезания (множитель)falseoutput_formatКонтейнер и кодек выходаmp4sample_durationДлительность сэмпла для тестирования (сек)300sample_offsetСмещение начала сэмпла от начала файла (сек)0

4. Режим 1 — CLI-скрипт
4.1 Поведение

Пользователь кладёт видеофайл(ы) в папку рядом со скриптом
Запускает run.bat / run.sh или напрямую python silencecut.py
Скрипт находит все видеофайлы в текущей директории (или принимает путь аргументом)
Обрабатывает каждый файл, создаёт рядом <имя_файла>_cut.<расширение>
Исходный файл не изменяется

4.2 Аргументы командной строки
python silencecut.py [OPTIONS] [FILE_OR_DIR]

Аргументы:
  FILE_OR_DIR              Путь к файлу или директории (по умолчанию: текущая папка)

Опции:
  --threshold FLOAT        Порог тишины в dB (например: -30)
  --duration FLOAT         Минимальная длительность тишины (сек)
  --padding-start INT      Отступ перед речью (мс)
  --padding-end INT        Отступ после речи (мс)
  --sample INT             Создать сэмпл длиной N секунд вместо полной обработки
  --sample-offset INT      Начало сэмпла (сек от начала файла)
  --output-dir PATH        Папка для результатов
  --dry-run                Только показать найденные сегменты, не обрабатывать
  --verbose                Подробный лог
4.3 Приоритет настроек
Командная строка > .env файл > значения по умолчанию
4.4 Формат .env
iniSILENCE_THRESHOLD=-30
SILENCE_DURATION=0.5
PADDING_START=100
PADDING_END=150
MIN_SEGMENT_DURATION=0.3
OUTPUT_FORMAT=mp4
SAMPLE_DURATION=300
SAMPLE_OFFSET=0
OUTPUT_SUFFIX=_cut
4.5 Логирование в CLI

Прогресс: [1/3] Processing: video.mp4
По завершении: найдено N сегментов тишины, вырезано X секунд, итоговая длина, путь к файлу
Режим --dry-run выводит таймкоды всех сегментов тишины без обработки

6.5 Обработка больших файлов в веб-режиме

Задачи выполняются асинхронно через очередь (asyncio или threading)
Прогресс парсится из stdout FFmpeg (time=HH:MM:SS) и транслируется клиенту через SSE (Server-Sent Events) или polling /status/{job_id} каждые 2 сек
При закрытии вкладки задача продолжает выполняться на сервере

8. Ядро обработки (silencecut.py)
8.1 Алгоритм

Анализ: Запуск ffmpeg -i input -af silencedetect=noise=THRESHOLD:d=DURATION -f null - — парсинг stdout для получения списка [{start, end}] тихих сегментов
Построение сегментов речи: Инвертирование тихих сегментов с учётом padding_start, padding_end, min_segment_duration
Нарезка и склейка:

Для каждого сегмента речи: ffmpeg -ss START -to END -i input -c copy segment_N.ts
Склейка через concat demuxer: ffmpeg -f concat -safe 0 -i list.txt -c copy output.mp4
Альтернатива для точных разрезов: filtergraph с select/aselect + setpts/asetpts (опция --accurate, медленнее но без переразметки)


Очистка: Удаление временных сегментов

8.2 Режим сэмпла
Перед основной обработкой извлекается фрагмент:
ffmpeg -ss OFFSET -t DURATION -i input -c copy sample_input.mp4
Далее обработка выполняется только для этого фрагмента.
8.3 Обработка ошибок

FFmpeg не найден → понятное сообщение с инструкцией по установке
Файл повреждён → ошибка с кодом FFmpeg и stderr
Недостаточно места на диске → проверка перед стартом (оценка: размер входного файла × 1.5)
Прерывание пользователем (Ctrl+C) → корректная очистка временных файлов


9. Нефункциональные требования

Производительность: Для анализа аудио используется только аудиопоток (-vn), что ускоряет парсинг тишины на больших файлах
Переносимость: Windows 10+, macOS 12+, Ubuntu 20.04+ без изменений кода
Зависимости: Минимальны. CLI-режим — только стандартная библиотека Python + subprocess. Веб-режим — FastAPI, Uvicorn, python-multipart
Кодировка файлов: Поддержка путей с кириллицей и пробелами
Логи: Все операции пишутся в silencecut.log рядом со скриптом
